# بوابة فضاء 👻 — استقبال طلبات العملاء قبل الاجتماع

بوابة بهوية فضاء يعبّيها العميل بأقل من دقيقة (٥ خطوات، كلها اختيارات تقريباً)،
والطلب يوصل الفريق مع بريف ذكي جاهز للاجتماع.

## المعمارية

```
العميل → index.html (GitHub Pages)
            ↓ كتابة مباشرة (قواعد create-only)
        Firestore: fada_leads          ← شغّال الآن ✅
            ↓ trigger عند كل طلب جديد
        fadaSupervisor (Cloud Function) ← جاهز، ينتظر ترقية Blaze ⏳
            ├─ Claude Haiku: بريف (ملخص + جدية + علامات حمراء + ٣ أسئلة للاجتماع)
            ├─ تحديث المستند بالبريف
            └─ إشعار تيليغرام للفريق
```

- **يشتغل اليوم بدون أي ترقية:** الطلبات تنحفظ في `fada_leads` وتشوفها من
  [كونسول Firestore](https://console.firebase.google.com/project/ahdaf-influencers/firestore/data/~2Ffada_leads).
- **المشرف الذكي (الـAI):** يحتاج خطة Blaze — انظر «تفعيل المشرف» تحت.

## الأمان
- قواعد Firestore تسمح **فقط** بإنشاء طلب في `fada_leads` (بتحقق من الحقول والأحجام) —
  لا قراءة ولا تعديل ولا حذف من المتصفح. كل الكوليكشنات الأخرى مقفولة.
- ملاحظة: القواعد التجريبية القديمة للمشروع انتهت 2026-07-04 (كان كل شيء مرفوضاً أصلاً).
  إذا تطبيق أهداف يحتاج وصول client من جديد، أضف له قواعد بمصادقة.

## تفعيل المشرف الذكي (مرة واحدة)
1. رقّ المشروع إلى Blaze: https://console.firebase.google.com/project/ahdaf-influencers/usage/details
   (الاستخدام هنا فعلياً داخل الحد المجاني — طلبات قليلة شهرياً)
2. (اختياري — إشعار تيليغرام) أنشئ بوت عبر @BotFather وأضفه لقروب الفريق، ثم:
   ```bash
   cd فضاء-بوابة/functions
   echo 'FADA_TELEGRAM_TOKEN=123:ABC...' >> .env
   echo 'FADA_TELEGRAM_CHAT=-100xxxxxxxx' >> .env
   ```
3. انشر:
   ```bash
   cd فضاء-بوابة
   firebase deploy --only functions:fada
   ```
   سر `ANTHROPIC_KEY` يُطلب عند أول نشر إن لم يكن موجوداً:
   `firebase functions:secrets:set ANTHROPIC_KEY`

## تعديل الواجهة
كل شيء في `index.html` (ملف واحد + `assets/`). بعد التعديل:
```bash
git add -A && git commit -m "تحديث" && git push
```
GitHub Pages يحدّث الرابط تلقائياً خلال دقيقة.

## قرارات من جلسة المجلس (llm-council)
- الفورم قصير عمداً: ٥ خطوات، أغلبها لمسات — أي سؤال زيادة = عميل ينسحب.
- الميزانية **نطاقات** + خيار «ما حددت بعد» (سؤال مفتوح = تجاهل أو رقم وهمي).
- مهرب واتساب في أول شاشة: «أرسل صوتية واشرح براحتك» — العميل اللي ما يحب الفورم ما نخسره.
- الـAI يولّد **أسئلة متابعة للاجتماع** مو بس تلخيص — عشان الاجتماع الأول يقفل مو يستكشف.
- ⚠️ تحذير المجلس: لو عبّى العميل الفورم ثم انسأل نفس الأسئلة بالاجتماع، انكسرت الثقة —
  الفريق لازم يقرأ البريف قبل ما يدخل.
- 📏 معيار القتل المتفق عليه: إذا ٣ من أول ٥ عملاء ما عبّوا — نوقف ونراجع النهج
  (البديل: أسئلة واتساب مباشرة والـAI يلخص المحادثة).
